# 前端 15173 公网入口设计

## 目标

允许用户通过 `http://服务器IP:15173` 访问 LongInvest 前端，并继续由前端 Nginx 将 `/api/` 请求转发到容器网络中的后端服务。

## 方案

- 将 Compose 的前端端口绑定从 `127.0.0.1:15173:8080` 改为 `15173:8080`。
- 保持前端容器内 Nginx 监听 `8080`，静态页面和 `/api/` 转发规则不变。
- 后端 `18080` 继续只绑定 `127.0.0.1`。
- PostgreSQL 和 Redis 继续不发布宿主机端口。
- 不修改服务器现有的 80、443 端口及其站点配置。

## 安全边界

公网只能直接访问前端 Nginx。后端、数据库和缓存仍位于内部网络；所有 API 请求继续经过前端 Nginx，保留现有请求来源、Cookie、CSRF 和登录校验。

## 部署与恢复

部署时只重建前端容器，不删除数据卷。若公网入口异常，将端口绑定恢复为 `127.0.0.1:15173:8080` 并重新创建前端容器即可回退。

## 验收标准

- 服务器监听 `0.0.0.0:15173` 或等价的全部 IPv4 地址。
- `http://服务器IP:15173` 返回前端页面。
- 经该入口访问 `/api/` 能到达后端，未登录请求按现有规则返回认证错误。
- API 的 `18080`、PostgreSQL 和 Redis 未新增公网监听。
- 前端容器健康，且没有反复重启。
